# Программа измерений проекта

## 1. Цель
Разработка системы метрик для оценки:
- эффективности процесса разработки (3 метрики);
- качества программного продукта (2 метрики).

Метрики сформированы по методике **GQ(I)M** (Goal–Question–Indicator–Metric) из материала лекции 6.

---

## 2. Метрики эффективности процесса разработки

### 2.1. Производительность
| Параметр | Значение                                                                                         |
|----------|--------------------------------------------------------------------------------------------------|
| **Цель** | Повышение производительности труда                                                               |
| **Вопрос** | Какой объём кода создаётся за единицу времени?                                                   |
| **Формула** | `Productivity = LOC / затраченное время (часы)`                                                  |
| **Единица измерения** | строк кода / человеко-часов                                                                      |
| **Индикатор** | Progress (отслеживание динамики)                                                                 |

### 2.2. Problem Resolution Rate (PRR)
| Параметр | Значение |
|----------|----------|
| **Цель** | Сокращение времени обработки задач |
| **Вопрос** | Сколько дней в среднем занимает выполнение одной задачи? |
| **Формула** | `PRR = Σ(дата закрытия − дата создания) / число закрытых задач` |
| **Единица измерения** | дней на задачу |
| **Индикатор** | Progress (снижение значения — положительный тренд) |

### 2.3. On Project % Time (OPPT)
| Параметр | Значение |
|----------|----------|
| **Цель** | Оптимизация распределения рабочего времени |
| **Вопрос** | Какую долю рабочего времени занимает работа по проекту? |
| **Формула** | `OPPT = (время по проекту / общее рабочее время) × 100%` |
| **Единица измерения** | % |
| **Индикатор** | Analysis (оценка загрузки и планирование) |

---

## 3. Метрики качества программного продукта

### 3.1. Product Fault Density – Coding (PFD-Coding)
| Параметр | Значение |
|----------|----------|
| **Цель** | Повышение качества кода |
| **Вопрос** | Какова плотность ошибок, допущенных на этапе кодирования? |
| **Формула** | `PFD-Coding = число ошибок в коде / LOC` |
| **Единица измерения** | ошибок на 1000 строк кода (KLOC) |
| **Как считать** | • Ошибки: закрытые баги с меткой "Bug"<br>• LOC: `cloc` на момент релиза |
| **Индикатор** | Success (целевое значение: снижение плотности) |

### 3.2. In Process Faults (IPF)
| Параметр | Значение |
|----------|----------|
| **Цель** | Контроль качества до выпуска релиза |
| **Вопрос** | Сколько ошибок обнаружено в продукте до его релиза? |
| **Формула** | `IPF = число ошибок, найденных до релиза / LOC` |
| **Единица измерения** | ошибок на 1000 строк кода (KLOC) |
| **Как считать** | • Ошибки: все закрытые баги до даты релиза<br>• LOC: `cloc` на дату релиза |
| **Индикатор** | Success (чем меньше — тем лучше) |
